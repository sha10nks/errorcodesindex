<p class="c1"><span class="c0">Title</span></p>
<p class="c1"><span class="c0">(M279 &mdash; Claim rejected)</span></p>
<p class="c1"><span class="c0">One-sentence summary</span></p>
<p class="c1"><span class="c0">This code indicates that the claim was rejected during processing.</span></p>
<p class="c1"><span class="c0">What this code means</span></p>
<p class="c1"><span class="c0">M279 is applied when CMS rejects a claim due to administrative or policy reasons. Rejection occurs before full adjudication. The code does not specify the reason by itself. Additional messages usually explain the rejection cause.</span></p>
<p class="c1"><span class="c0">Where users usually see this code</span></p>
<p class="c1"><span class="c0">Medicare remittance advice</span></p>
<p class="c1"><span class="c0">Claim rejection notices</span></p>
<p class="c1"><span class="c0">Submission status reports</span></p>
<p class="c1"><span class="c0">Why this code usually appears</span></p>
<p class="c1"><span class="c0">Submission errors detected</span></p>
<p class="c1"><span class="c0">Policy or eligibility checks failed</span></p>
<p class="c1"><span class="c0">Required data missing</span></p>
<p class="c1"><span class="c0">What typically happens next</span></p>
<p class="c1"><span class="c0">Claim processing stops</span></p>
<p class="c1"><span class="c0">Reprocessing may be required</span></p>
<p class="c1"><span class="c0">Payment is not issued</span></p>
<p class="c1"><span class="c0">What this code is NOT</span></p>
<p class="c1"><span class="c0">It is not a specific denial reason</span></p>
<p class="c1"><span class="c0">It is not patient responsibility</span></p>
<p class="c1"><span class="c0">It is not a contractual adjustment</span></p>
<p class="c1"><span class="c0">Related error codes</span></p>

<p class="c1"><span class="c0">Notes and edge cases</span></p>
<p class="c1"><span class="c0">Rejected claims differ from denied claims. Rejection often allows correction and resubmission.</span></p>
