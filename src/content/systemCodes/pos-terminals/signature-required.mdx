<p>SECTION: Systems &amp; Devices</p>
<p>SYSTEM: POS Terminals</p>
<p>CODE: SIGNATURE</p>
<p>Title</p>
<p>(SIGNATURE &mdash; Signature required)</p>
<p>One-sentence summary</p>
<p>The transaction is approved or conditionally approved but requires signature verification per the processing rules.</p>
<p>What this code means</p>
<p>Signature prompts occur when the processing rules for the card/product/transaction type indicate signature capture is required. The transaction may already be approved, but completion requires the merchant to follow the verification step.</p>
<p>Where users usually see this code</p>
<p>Card-present approvals on certain card products</p>
<p>Offline or fallback scenarios (processor-dependent)</p>
<p>Why this code usually appears</p>
<p>Card network rules require signature</p>
<p>Terminal is configured to prompt for signature on certain amounts</p>
<p>What typically happens next</p>
<p>Merchant captures signature and completes the sale</p>
<p>What this code is NOT</p>
<p>It is not a decline reason</p>
<p>It is not a terminal malfunction</p>
<p>Troubleshooting checklist</p>
<p>Follow on-screen prompts and capture the signature if required</p>
<p>Keep receipt records per your processor/acquirer policy</p>
<p>Related error codes</p>
<p>CALL ISSUER — Issuer requests contact</p>
<p>DECLINED — Authorization declined</p>
<p>Notes and edge cases</p>
<p>Signature requirements vary by network and region; rely on your processor’s configured rules.</p>
