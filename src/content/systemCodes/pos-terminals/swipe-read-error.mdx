<p>SECTION: Systems &amp; Devices</p>
<p>SYSTEM: POS Terminals</p>
<p>CODE: SWIPE READ</p>
<p>Title</p>
<p>(SWIPE READ &mdash; Magstripe read error)</p>
<p>One-sentence summary</p>
<p>The terminal could not read the magstripe data from the card during a swipe.</p>
<p>What this code means</p>
<p>Magstripe read errors occur when the swipe data is incomplete or cannot be decoded. Common causes include damaged stripes, dirty readers, or improper swipe speed.</p>
<p>Where users usually see this code</p>
<p>Swipe transactions when chip/tap is unavailable</p>
<p>Older cards with worn magstripes</p>
<p>Why this code usually appears</p>
<p>Magstripe is scratched or demagnetized</p>
<p>Reader is dirty or worn</p>
<p>Swipe speed/angle caused incomplete read</p>
<p>What typically happens next</p>
<p>Terminal requests another swipe or different payment method</p>
<p>What this code is NOT</p>
<p>It is not a decline decision by the issuer</p>
<p>It is not a batch close error</p>
<p>Troubleshooting checklist</p>
<p>Retry swipe with steady speed and correct orientation</p>
<p>Clean the reader per vendor guidance</p>
<p>Use chip/tap if possible</p>
<p>Related error codes</p>
<p>CHIP READ — Chip read error</p>
<p>DECLINED — Authorization declined</p>
<p>Notes and edge cases</p>
<p>Many merchants restrict magstripe fallback for EMV cards; follow processor rules.</p>
