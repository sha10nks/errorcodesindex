<p>SECTION: Systems &amp; Devices</p>
<p>SYSTEM: BIOS / UEFI</p>
<p>CODE: REBOOT AND SELECT PROPER BOOT DEVICE</p>
<p>Title</p>
<p>(REBOOT AND SELECT PROPER BOOT DEVICE &mdash; Wrong boot device)</p>
<p>One-sentence summary</p>
<p>Firmware cannot find a bootable device from the current boot order selection.</p>
<p>What this code means</p>
<p>This message indicates the system did not find a bootable OS on the selected device and is prompting you to choose a proper boot device. It is commonly caused by boot order changes, missing drives, or non-bootable media.</p>
<p>Where users usually see this code</p>
<p>After adding/removing drives or USB devices</p>
<p>After BIOS/UEFI reset to defaults</p>
<p>Why this code usually appears</p>
<p>Boot order points to an empty/non-bootable device</p>
<p>System drive is not detected</p>
<p>Boot configuration is missing or corrupted</p>
<p>What typically happens next</p>
<p>The system will not boot until boot order/device is corrected</p>
<p>What this code is NOT</p>
<p>It is not an application error</p>
<p>It is not a Windows Update code</p>
<p>Troubleshooting checklist</p>
<p>Remove external media and retry</p>
<p>Enter BIOS/UEFI and select the correct boot drive</p>
<p>Check drive detection and cables (desktop systems)</p>
<p>Related error codes</p>
<p>NO BOOTABLE DEVICE — Boot device not found</p>
<p>OPERATING SYSTEM NOT FOUND — No OS detected</p>
<p>Notes and edge cases</p>
<p>After firmware updates/resets, Secure Boot and boot mode changes can alter boot behavior.</p>
