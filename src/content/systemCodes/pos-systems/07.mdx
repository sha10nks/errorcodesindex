<p>SECTION: Systems &amp; Devices</p>
<p>SYSTEM: POS Systems</p>
<p>CODE: 07</p>
<p>Title</p>
<p>(07 &mdash; Pick up card (special condition))</p>
<p>One-sentence summary</p>
<p>The issuer indicates a retain/pick-up action under special conditions; the transaction should not proceed.</p>
<p>What this code means</p>
<p>Response code 07 is a “retain card” style response that indicates the issuer considers the transaction high risk or the card status problematic. Many merchants are not able to retain cards; it is commonly handled as a hard decline.</p>
<p>Where users usually see this code</p>
<p>High-risk or unusual purchase attempts</p>
<p>Transactions with issuer risk blocks</p>
<p>Why this code usually appears</p>
<p>Issuer fraud/risk controls triggered</p>
<p>Card status flagged at the issuer level</p>
<p>What typically happens next</p>
<p>The transaction is declined</p>
<p>Cardholder is advised to contact their issuer</p>
<p>What this code is NOT</p>
<p>It is not a terminal hardware failure</p>
<p>It is not a merchant configuration issue</p>
<p>Troubleshooting checklist</p>
<p>Do not retry repeatedly; request another payment method</p>
<p>Follow merchant/acquirer safety policy regarding retain instructions</p>
<p>Related error codes</p>
<p>04 — Pick up card</p>
<p>41 — Lost card</p>
<p>43 — Stolen card</p>
<p>Notes and edge cases</p>
<p>Operational handling differs by region and merchant category; prioritize staff safety.</p>
