<p>SECTION: Systems &amp; Devices</p>
<p>SYSTEM: POS Systems</p>
<p>CODE: 41</p>
<p>Title</p>
<p>(41 &mdash; Lost card)</p>
<p>One-sentence summary</p>
<p>The issuer indicates the card is reported lost; the authorization should be declined.</p>
<p>What this code means</p>
<p>Response code 41 indicates the issuer has flagged the card as lost. Transactions should not proceed, and the customer must contact their issuer. Operational handling of card-retain instructions depends on merchant and acquirer policy.</p>
<p>Where users usually see this code</p>
<p>Authorization attempts on a card reported lost</p>
<p>Terminal messages indicating a hard decline</p>
<p>Why this code usually appears</p>
<p>The cardholder reported the card lost</p>
<p>The issuer blocked the account/card for security reasons</p>
<p>What typically happens next</p>
<p>The transaction is declined</p>
<p>The customer uses a different payment method</p>
<p>What this code is NOT</p>
<p>It is not a connectivity problem</p>
<p>It is not a merchant setup error</p>
<p>Troubleshooting checklist</p>
<p>Ask the customer to contact their issuer and use another payment method</p>
<p>Follow acquirer guidance for any card-retain messaging</p>
<p>Related error codes</p>
<p>43 — Stolen card</p>
<p>04 — Pick up card</p>
<p>05 — Do not honor</p>
<p>Notes and edge cases</p>
<p>Merchants typically should not take unsafe actions; follow your acquirer/merchant handbook.</p>
