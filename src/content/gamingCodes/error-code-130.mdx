<p class="c1"><span class="c0">Title</span></p>
<p class="c1"><span class="c0">(Error Code 130 &mdash; Server error)</span></p>
<p class="c1"><span class="c0">One-sentence summary</span></p>
<p class="c1"><span class="c0">This code indicates that a server-side error prevented a request from completing.</span></p>
<p class="c1"><span class="c0">What this code means</span></p>
<p class="c1"><span class="c0">Error Code 130 appears when a server encounters an internal problem. The issue originates from backend systems. It does not indicate client or account faults. Requests cannot be processed until the server stabilizes. Multiple users may be affected.</span></p>
<p class="c1"><span class="c0">Where users usually see this code</span></p>
<p class="c1"><span class="c0">Login or matchmaking attempts</span></p>
<p class="c1"><span class="c0">Online feature access</span></p>
<p class="c1"><span class="c0">Service request failures</span></p>
<p class="c1"><span class="c0">Why this code usually appears</span></p>
<p class="c1"><span class="c0">Backend processing error</span></p>
<p class="c1"><span class="c0">Temporary infrastructure instability</span></p>
<p class="c1"><span class="c0">Service component failure</span></p>
<p class="c1"><span class="c0">What typically happens next</span></p>
<p class="c1"><span class="c0">Request fails</span></p>
<p class="c1"><span class="c0">Online features remain unavailable</span></p>
<p class="c1"><span class="c0">Access restores after recovery</span></p>
<p class="c1"><span class="c0">What this code is NOT</span></p>
<p class="c1"><span class="c0">It is not a client configuration issue</span></p>
<p class="c1"><span class="c0">It is not a license error</span></p>
<p class="c1"><span class="c0">It is not a network misconfiguration</span></p>
<p class="c1"><span class="c0">Related error codes</span></p>
<p class="c1"><span class="c0">Error Code 500 &mdash; Internal server error</span></p>
<p class="c1"><span class="c0">Error Code 503 &mdash; Service unavailable</span></p>
<p class="c1"><span class="c0">Notes and edge cases</span></p>
<p class="c1"><span class="c0">Server errors may be brief or extended. Recovery depends on backend systems.</span></p>
<p class="c1"><span class="c0">Optional next actions</span></p>
<p class="c1"><span class="c0">Retry the request later</span></p>
<p class="c1"><span class="c0">Monitor service updates</span></p>
<p class="c1"><span class="c0">Observe recovery behavior</span></p>
<p class="c1"><span class="c0">External help (optional)</span></p>
<p class="c1"><span class="c0">Service status channels may provide server availability information.</span></p>
