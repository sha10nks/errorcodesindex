<p class="c1"><span class="c0">Title</span></p>
<p class="c1"><span class="c0">(Error Code 800 &mdash; Version mismatch)</span></p>
<p class="c1"><span class="c0">One-sentence summary</span></p>
<p class="c1"><span class="c0">This code indicates that the client version does not match the service version.</span></p>
<p class="c1"><span class="c0">What this code means</span></p>
<p class="c1"><span class="c0">Error Code 800 appears when the client and server versions are incompatible. The service rejects requests from mismatched versions. This prevents inconsistent behavior. It does not indicate account enforcement. Synchronization is required before access resumes.</span></p>
<p class="c1"><span class="c0">Where users usually see this code</span></p>
<p class="c1"><span class="c0">Login attempts</span></p>
<p class="c1"><span class="c0">Online feature access</span></p>
<p class="c1"><span class="c0">Matchmaking or session creation</span></p>
<p class="c1"><span class="c0">Why this code usually appears</span></p>
<p class="c1"><span class="c0">Client and server version mismatch</span></p>
<p class="c1"><span class="c0">Partial update deployment</span></p>
<p class="c1"><span class="c0">Version synchronization delay</span></p>
<p class="c1"><span class="c0">What typically happens next</span></p>
<p class="c1"><span class="c0">Access remains blocked</span></p>
<p class="c1"><span class="c0">Retry may fail until versions align</span></p>
<p class="c1"><span class="c0">Access restores after sync</span></p>
<p class="c1"><span class="c0">What this code is NOT</span></p>
<p class="c1"><span class="c0">It is not a network outage</span></p>
<p class="c1"><span class="c0">It is not a server crash</span></p>
<p class="c1"><span class="c0">It is not a ban</span></p>

<p class="c1"><span class="c0">Version mismatches often occur during staggered updates. Resolution may be automatic.</span></p>
