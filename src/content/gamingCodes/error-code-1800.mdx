<p class="c1"><span class="c0">Title</span></p>
<p class="c1"><span class="c0">(Error Code 1800 &mdash; Region restricted)</span></p>
<p class="c1"><span class="c0">One-sentence summary</span></p>
<p class="c1"><span class="c0">This code indicates that access is restricted based on region.</span></p>
<p class="c1"><span class="c0">What this code means</span></p>
<p class="c1"><span class="c0">Error Code 1800 appears when a service blocks access due to geographic restrictions. The restriction is enforced by policy. It does not indicate a system error. Access is blocked regardless of account status. Availability depends on location rules.</span></p>
<p class="c1"><span class="c0">Where users usually see this code</span></p>
<p class="c1"><span class="c0">Content access attempts</span></p>
<p class="c1"><span class="c0">Service login screens</span></p>
<p class="c1"><span class="c0">Feature availability checks</span></p>

<p class="c1"><span class="c0">Regional availability rules enforced</span></p>
<p class="c1"><span class="c0">Licensing or policy restrictions</span></p>
<p class="c1"><span class="c0">Location-based access control</span></p>
<p class="c1"><span class="c0">What typically happens next</span></p>
<p class="c1"><span class="c0">Access remains blocked</span></p>
<p class="c1"><span class="c0">Content unavailable</span></p>
<p class="c1"><span class="c0">Status unchanged</span></p>
<p class="c1"><span class="c0">What this code is NOT</span></p>
<p class="c1"><span class="c0">It is not a network failure</span></p>
<p class="c1"><span class="c0">It is not an account ban</span></p>
<p class="c1"><span class="c0">It is not a server outage</span></p>
