<p class="c1"><span class="c0">Title</span></p>
<p class="c1"><span class="c0">(150 &mdash; Return filed)</span></p>
<p class="c1"><span class="c0">One-sentence summary</span></p>
<p class="c1"><span class="c0">This code indicates the IRS has received and recorded the tax return as filed in its system.</span></p>
<p class="c1"><span class="c0">What this code means</span></p>
<p class="c1"><span class="c0">Code 150 appears on an IRS account transcript to show that a tax return has been filed and entered into the IRS processing system. It represents the official recording of the tax liability as reported on the return. This code is typically one of the first transaction codes posted for a given tax period and reflects that the return has moved beyond simple receipt into formal account processing. It does not indicate approval of a refund, completion of processing, or absence of review. It simply confirms the return has been posted to the taxpayer&rsquo;s account record for that year.</span></p>
<p class="c1"><span class="c0">Where users usually see this code</span></p>
<p class="c1"><span class="c0">On an IRS account transcript under the transaction codes section</span></p>
<p class="c1"><span class="c0">When reviewing tax account history for a specific tax year</span></p>
<p class="c1"><span class="c0">After a return has been accepted and entered into processing</span></p>
<p class="c1"><span class="c0">Why this code usually appears</span></p>
<p class="c1"><span class="c0">The IRS has accepted the return and posted it to the taxpayer&rsquo;s account</span></p>
<p class="c1"><span class="c0">The tax liability reported on the return has been recorded</span></p>
<p class="c1"><span class="c0">The return completed initial intake validation</span></p>
<p class="c1"><span class="c0">Processing moved from submission to account-level recording</span></p>
<p class="c1"><span class="c0">What typically happens next</span></p>
<p class="c1"><span class="c0">Additional transaction codes may appear reflecting credits, adjustments, or refund processing</span></p>
<p class="c1"><span class="c0">The account may update with refund-related codes if applicable</span></p>
<p class="c1"><span class="c0">The return may move through standard processing or review workflows</span></p>
<p class="c1"><span class="c0">What this code is NOT</span></p>
<p class="c1"><span class="c0">It is not a refund approval notice</span></p>
<p class="c1"><span class="c0">It is not confirmation that the return has completed all review stages</span></p>
<p class="c1"><span class="c0">It is not an indication of audit or examination by itself</span></p>

<p class="c1"><span class="c0">Notes and edge cases</span></p>
<p class="c1"><span class="c0">Code 150 often appears with a cycle date that reflects the IRS processing week. The presence of this code does not indicate whether a refund will be issued or delayed. In some cases, additional transaction codes may appear shortly after 150 that reflect credits, offsets, or holds. The timing between code 150 and later updates can vary depending on filing method, workload, and review requirements.</span></p>
